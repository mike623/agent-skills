# Application submission co-pilot notes

Use when moving from shortlisted roles to actual applications.

## Durable lesson

Many UK recruiter/job-board applications cannot be fully completed autonomously because they require account creation, login, magic links, passwords, salary/status declarations, or consent checkboxes. Treat submission as a co-pilot workflow unless the user has already provided the non-secret fields and is present for login/password steps.

## Before attempting submissions

Collect or confirm the fields that commonly block forms:

- current salary or salary expectation;
- notice period / availability;
- UK work status / visa sponsorship requirement;
- right-to-work location constraints;
- driving licence / own transport where asked;
- whether the user wants recruiter consent/contact boxes selected;
- preferred phone/email to submit;
- CV file path and whether cover note should be tailored per role.

Never type, generate, store, or ask the user to reveal passwords. If an application requires account setup, bring the user to that point and ask them to complete the secret/login step themselves.

## Practical fallback hierarchy

1. Prefer direct employer/ATS apply pages when accessible.
2. If aggregator pages block automation or require login, look for recruiter mirrors and direct recruiter email.
3. If a recruiter email is available, prepare a concise tailored email + CV attachment path as the fastest low-friction application route.
4. If only login-gated portals exist, prepare an application pack and tell the user exactly what they must complete manually.

## Reporting pattern

Avoid surprising the user with "you need to apply yourself" after a long attempt. State early:

- what can be automated: shortlist, CV/cover note tailoring, field prefill, application pack, email drafts;
- what needs the user: passwords, magic links, personally attested declarations, CAPTCHA/payment/permission prompts;
- exact artifacts prepared: CV path, notes path, role URLs, recruiter emails, and remaining manual steps.

## Example concise status

"I can co-pilot these applications, but not fully submit login-gated forms without you. I prepared the CV and tailored notes. Direct-email applications can be sent fastest; login/password steps need you present."
